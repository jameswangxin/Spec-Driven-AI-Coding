# REQ-0001 技术方案

## 1. 目标

增加后台报表 CSV 导出能力，并确保权限、空结果和失败处理可验证。

## 2. 非目标

- 不支持 Excel。
- 不支持异步导出。
- 不支持邮件发送。

## 3. 改动范围

### 文件 / 模块

- 报表接口。
- 导出服务。
- 权限校验。
- 测试用例。

### 行为变化

- 后台报表页增加导出能力。

### 影响对象

- 运营用户。
- 后台服务。

## 4. 方案设计

在报表查询基础上增加导出入口。导出前复用报表权限校验，查询结果后生成 CSV 并返回下载。

## 5. 数据流 / 调用链路

```text
Controller
  -> Permission Check
  -> Report Query
  -> CSV Builder
  -> Response
```

## 6. 接口 / 数据结构变化

### 新增

- `GET /reports/export`

### 修改

- 无。

### 删除

- 无。

## 7. 兼容性与迁移

- 无数据迁移。

## 8. 风险与回滚

| 风险 | 影响 | 缓解方式 |
| --- | --- | --- |
| 导出数据过大 | 响应变慢 | 限制默认时间范围 |
| 权限遗漏 | 数据泄露 | 复用查询权限校验 |

回滚方式：

- 下线导出入口。

## 9. 测试策略

### 自动化测试

- 有权限导出成功。
- 无权限导出失败。
- 空结果导出只有表头。

### 手工验证

- 在后台页面点击导出。

### 不覆盖项与原因

- 不覆盖超大数据异步导出，本需求不支持。

## 10. 实施步骤

1. 编写权限测试。
2. 编写导出成功测试。
3. 实现导出接口。
4. 实现 CSV 生成。
5. 运行测试并记录结果。
