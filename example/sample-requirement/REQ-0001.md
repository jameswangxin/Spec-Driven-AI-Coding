# REQ-0001：增加导出报表功能

## 原始需求

用户希望在后台增加报表导出能力。

## 背景与目标

运营人员需要将筛选后的报表导出为 CSV，用于线下分析。

## 关联范围

### 相关模块

- 报表查询。
- 导出任务。

### 相关文件 / 接口 / 数据表

- 待目标项目填写。

### 关联需求 / 能力规格

- 无。

## 业务规则

| 场景 | 规则 | 备注 |
| --- | --- | --- |
| 无筛选条件 | 默认导出最近 30 天 | 避免全量导出 |
| 无权限用户 | 禁止导出 | 记录审计日志 |
| 数据为空 | 生成空 CSV | 保留表头 |

## 边界条件

### 做什么

- 支持 CSV 导出。
- 支持权限校验。
- 支持空结果导出。

### 不做什么

- 不支持 Excel。
- 不支持异步邮件发送。

### 异常情况

- 导出失败时返回明确错误，并记录日志。

## 需求确认

### 术语定义

| 术语 | 定义 |
| --- | --- |
| 报表 | 后台统计列表 |
| 导出 | 将当前筛选结果生成 CSV |

### 数据流 / 调用链路

```text
用户点击导出
  -> 权限校验
  -> 查询报表数据
  -> 生成 CSV
  -> 返回下载
```

### 权限 / 安全 / 数据一致性

- 权限：只允许有报表权限的用户导出。
- 安全：导出内容不能包含脱敏字段。
- 数据一致性：导出结果以请求时查询条件为准。

### 待确认问题

- 无。

## 验收标准

- 有权限用户可以导出 CSV。
- 无权限用户不能导出。
- 空数据时返回只有表头的 CSV。

## 测试用例

### 正常场景

- 有筛选条件时导出对应数据。

### 边界场景

- 无数据时导出空 CSV。

### 异常场景

- 无权限时返回错误。
